# Plume AI Export Format (v1)

A JSON serialization of a Plume vault optimised for fast consumption by
AI agents and other tools. Generated by **Settings → Export for AI**, or
programmatically via `buildAIExport()` in `src/store.ts`.

The format is _derived_ — it never replaces the underlying notes (which
stay in IndexedDB / localStorage). Re-exporting always reflects the
current state.

## Top-level shape

```jsonc
{
  "format": "plume.ai-export.v1",
  "exportedAt": "2026-05-06T01:23:45.000Z",
  "app": { "name": "Plume", "version": "1.0" },

  "stats": {
    "notes": 42,
    "folders": 5,
    "tags": 7,
    "approxTokensTotal": 18342,
    "wikilinks": 87
  },

  "folders":  [ { "id": "...", "name": "Work", "color": "#f59e0b", "parentId": null } ],
  "tags":     [ { "id": "...", "name": "idea", "color": "#22c55e", "noteCount": 12 } ],
  "notes":    [ /* PlumeAINote[] — see below */ ],

  "graph": {
    "nodes": [ { "id": "<note-id>", "title": "Hello world" } ],
    "edges": [ { "from": "<src-note-id>", "to": "<dst-note-id>" } ]
  }
}
```

## `PlumeAINote`

```ts
interface PlumeAINote {
  id: string
  title: string
  folder: string | null     // resolved folder name (not id)
  tags: string[]            // resolved tag names (not ids)
  createdAt: string         // ISO 8601
  updatedAt: string
  isPinned: boolean
  isArchived: boolean
  content: string           // raw markdown source (verbatim)
  approxTokens: number      // heuristic: ceil(content.length / 4)
  outbound: string[]        // [[Target]] titles found in content (deduped)
  inbound: string[]         // titles of other notes that link here
  snippet: string           // first 280 chars of stripped plain text
}
```

## Why this format

Plain Markdown round-tripping is fine for editing but slow to consume
for an agent that just wants to know "who links to X?" or "give me the
short summary of the active note's neighbourhood". This export bakes in:

- **Snippets** so a top-of-prompt overview can fit many notes.
- **Inbound + outbound wikilinks** pre-resolved by id so an agent can
  walk the graph without re-parsing every note.
- **`approxTokens`** so a context-budget can be planned cheaply (4
  chars / token is the standard rough heuristic).
- **`stats`** at the top so a small probe request can decide whether
  to fetch the full payload at all.
- **Resolved names** for `folder` and `tags` — agents do not have to
  hold a separate id-to-name map.

## Stability

`format: "plume.ai-export.v1"` is the locked v1 contract:

- Adding new keys is non-breaking. Consumers should ignore unknown
  fields.
- Removing or renaming any current field bumps to `v2` (`format:
  "plume.ai-export.v2"`).
- `id`s are opaque strings — never parse them.
- `inbound` / `outbound` are arrays of titles _as written by the user_,
  case-preserved. Lookups should be case-insensitive.

## Suggested agent prompt fragment

```
The user has attached a plume.ai-export.v1 JSON file.
- Use `notes[].snippet` for fast top-down browsing.
- Use `graph.edges` to find related notes by id.
- `notes[].outbound` and `notes[].inbound` are precomputed if you want
  to walk the wikilink graph by titles.
- `stats.approxTokensTotal` lets you check whether the whole vault
  fits in your context, or whether you should pull notes by id from
  `notes[]` on demand.
```
